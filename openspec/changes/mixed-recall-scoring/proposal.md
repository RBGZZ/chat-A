## Why

当前召回排序只用 `score = importance × decay`(记忆强度单路信号),关键词命中度完全没进打分——多关键词全命中和只命中一个的记忆在排序上无差别。canonical §5.5 要求**混合召回**:把关键词分(FTS bm25 风格)、记忆强度、情感共振融合成单一得分,且要解决两个工程坑:

1. **关键词原始分无界**:LIKE 命中数(或未来 FTS5 `bm25()`)是无界原始分,直接加权会压垮 [0,1] 的其它信号。mem0 `scoring.py` 用**查询长度自适应 sigmoid** 把它压到 [0,1]。
2. **信号缺席被稀释**:固定分母(总是除以"满分信号数")会让"无情感/无关键词"的记忆得分被人为压低(被不存在的信号稀释)。需要**自适应分母**:某路信号缺席时分母自动缩小。

同时要避开 mem0 的反面教训:它用**语义门控硬丢**低分项,会丢"语义不相关但情感强共振"的记忆。chat-A 的门控 MUST 只对"零信号"生效——关键词/情感**单路**也能把记忆拉进候选池。

本切片**不含向量**(P2 另说);只把现有"关键词候选 + 记忆强度排序"升级成"关键词归一 + 自适应分母混合 + 可选情感共振重排",全部收敛进 `config.ts` 的**单一权威公式**(§3.2),两 store 调用点同步,`recall` 对外签名/返回类型不变(情感共振经**可选**入参,默认不启用)。

## What Changes

- **关键词原始分归一**:LIKE 命中 token 数 `raw` 经**查询长度自适应 sigmoid** `1/(1+exp(-s·(raw-m)))` 压到 [0,1](陡度 `s`、中点 `m` 外置为配置,`m` 随查询 token 数自适应)。
- **混合打分自适应分母**:`score = min(Σ 在场信号 / 在场信号数, 1)`——记忆强度恒在场;关键词在场(有 token 时);情感共振仅当调用方传入 PAD 时在场。某路缺席时分母自动缩,不被稀释。
- **零信号门控**:候选**仅当全部在场信号都为 0** 时才被丢弃;关键词或情感**单路非零**即可进候选池(不学 mem0 语义门控硬丢)。
- **情感共振重排(可选)**:`recall` 增**可选** PAD 入参,缺省不启用。启用时用 PAD/Russell 扇区**常量矩阵**(O(1) 查表)对候选算情感共振分,作为一路在场信号融入混合得分。
- 衰减/重要性继续用现有单一权威公式(`decayFactor`/`reinforceImportance`/`recallScore`)作为"记忆强度"那一路信号融入混合式。

非破坏性:`recall` 公共签名向后兼容(PAD 为可选末位入参,默认不启用情感共振);`MemoryRecord` 不加字段;两 store 行为一致由共享契约测试锁定。

## Impact

- **影响 canonical 章节**:§5.5(混合召回/打分归一/情感共振矩阵)、§3.2(单一权威公式 + 行为即配置)。与权威设计一致,无冲突。
- **代码**:**仅** `packages/memory`——`config.ts`(新增归一/混合/情感共振单一权威纯函数 + 配置项)、`sqlite-store.ts` 与 `in-memory-store.ts`(召回调用点同步,关键词命中数从候选过滤里带出)、`types.ts`(PAD 形状的本地最小类型,纯加法导出)。**不改** runtime/cognition/persona/providers/client/observability。
- **接缝/签名**:`recall` 对外签名仅**可选**追加 PAD 入参(默认不启用情感共振),返回类型不变 → cognition/runtime 现有调用零改动。

- **不涉及**:向量/语义召回(P2)、FTS5 真表(本期仍用 LIKE 候选 + JS 归一,FTS5 落地时只换关键词原始分来源,归一/混合式不变)、写路径与迁移(无 schema 变更)。
