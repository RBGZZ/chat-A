## 一、总纲：四角色架构

具身智能 = **感知 + 思考 + 想象 + 行动** 四件事，对应四个角色。主理人最初的设想是"Agent(思考大脑) + VLA(行动小脑) + 世界模型(外界感知)"，**经讨论校准为**：

```
        ┌─────────── 感知层（眼/耳）──────────────┐
        │  摄像头/麦 + 小模型 → 当前状态向量         │  ← 输入总线（PerceptionPort）
        └──────────────────┬───────────────────────┘
                           ↓
   ┌──────────────────────────────────────────────────┐
   │  Agent = 大脑(思考)        世界模型 = 想象/预演      │  ← System 2（慢）
   │  人格·记忆·决策·规划   ⇄   "这样做会怎样"+安全护栏    │
   └───────────────────────┬──────────────────────────┘
                           ↓ 下发意图（自然语言/结构化，非 latent）
   ┌──────────────────────────────────────────────────┐
   │  VLA 动作专家 = 小脑(运动协调)  flow-matching→执行器  │  ← System 1（快）
   └──────────────────────────────────────────────────┘
```

**关键校准（务必记牢）**：

1. **世界模型 ≠ 感知**。感知回答"现在是什么"；世界模型回答"如果我做动作 a，接下来会怎样"（action-conditioned 前向预测）。一个更强的 VLM ≠ 拥有世界模型。
2. **世界模型是大脑的子能力，不是第四个感官**。它夹在感知与决策之间，给大脑做"预演"。
3. **VLA 要"砍头"用**。完整 VLA 自带 System 2（VLM 规划），但 chat-A 的 Agent 已经是大脑——具身时只用 VLA 的 **System 1 动作专家**，条件输入来自 Agent 下发的意图，避免"两个大脑抢方向盘"。
4. **chat-A 有两个世界模型**：**社会世界模型**（建模用户心理/关系，现在就做）与**物理世界模型**（建模环境动力学，远期）。详见 §三。

## 二、世界模型讲透（补齐认知缺口）

### 2.1 三大技术流派

| 流派 | 代表 | 本质 | 6GB/树莓派可行性 | 对 chat-A |
|------|------|------|------------------|-----------|
| **隐空间动力学（MBRL）** | DreamerV3、TD-MPC2、IRIS | 压成 latent，脑内想象 rollout + MPC 规划 | ✅ **唯一现实档**——世界模型本体仅几 M~几十 M 参数，瓶颈在视觉编码而非动力学 | 远期物理世界模型的接缝标杆 |
| **JEPA（表征预测）** | I-JEPA → V-JEPA → **V-JEPA 2**（Meta 2025） | 不预测像素，预测未来的抽象 embedding（LeCun 路线） | ⚠️ 编码器 300M 勉强推理；完整 AC+MPC 偏重 | **当前最强开源 action-conditioned 世界模型**，62h 机器人视频微调即可真机零样本抓取，值得跟踪 |
| **生成式视频** | Genie 3、NVIDIA Cosmos、Sora | 直接在像素空间生成未来视频当模拟器 | ❌ Cosmos 2B 就要 32GB，数据中心级；Genie 3 闭源 | 仅远期当**云端合成数据/仿真器**，不是端上世界模型 |

### 2.2 两条必须记住的结论

- **嵌入式世界模型 = "小而专的隐空间动力学"，不是"大而通的视频生成"。** Cosmos/Genie/Sora 的炫酷 demo 不要带偏硬件预算与预期。
- **"能生成逼真视频 ≠ 理解物理"**（LeCun 原话；PhyWorld 等 2026 实证反复确认）。生成式世界模型当训练数据/仿真器用很好，当实时控制后端不可靠。
- **"零样本 Sim2Real" 对 Dreamer 类脆弱**：latent 动力学从特定环境拟合，reality gap 会让想象系统性偏离真实（实测 DreamerV3 零样本上真车每圈冲出赛道）。现实做法是 DayDreamer 式真机在线学习，或域随机化（代价是指数级样本）。

## 三、对 chat-A 的世界模型优先级

### 3.1 社会世界模型 —— 现在就投入（纯软件、零硬件、正中北极星）

陪伴产品的世界模型红利在**社会维度**，不在物理维度。形态不是 ViT/diffusion，而是 **Theory-of-Mind + 关系状态动力学**：建模"用户当前相信什么/想要什么/情绪如何"，预测"如果小雪此刻打断/反对/共情，关系和情绪会怎么走"——这正是世界模型的 action-conditioned 本质，只是状态空间是社会/心理。

- 与 LLM 同构，可直接落到现有 LLM 认知层 + PAD 情绪 + 记忆层之上的一个**轻量预测模块**。零新硬件、零显存压力。
- 学术锚点：Social World Models、ToMAgent（"会主动、会反对、长期伴侣"的北极星高度吻合）。
- **这是近期重点。**

### 3.2 物理世界模型 —— 远期，只做接缝预留 + 技术跟踪

- 现在该跟踪：**DreamerV3 / TD-MPC2**（嵌入式可跑的隐空间 MBRL 标杆）、**V-JEPA 2**（最先进开源 action-conditioned，LeCun 押注方向）。
- 现在不该碰：Cosmos / Genie 3（数据中心级，与嵌入式矛盾）。
- 真机阶段范式参考：**DayDreamer**（真机在线学习 + 学习/执行线程解耦满足延迟），与 chat-A 的"延迟预算/非阻塞/优雅降级"原则天然契合。

### 3.3 定调：本地世界模型 = **符号信念态 WM**（不是像素 WM），且与感知彻底分离

**前提决策（2026-06-28）**：现阶段**多模态感知留云端 API，本地不跑视觉模型**；本地保留给世界模型（"本地世界模型之后会部署"，现有一台 **3060 6GB** 作 PC 层验证机/部署目标，≠树莓派层，双层用 Factory 接缝分别选后端）。

**连带的硬结论**：

1. **本地 WM 是"符号信念态世界模型"，不是像素 Dreamer/JEPA。** 因为本地没有视觉编码器，WM 没有像素可编码 → 它只能在**已被云端感知好的符号信念态**上学动力学（state-based TD-MPC2/Dreamer，几 M 参数，6GB 富余，甚至可下放边缘）。这恰好就是 §3.1 的**社会世界模型**形态：预测"我这么回应，用户情绪/关系怎么走"。
2. **感知与 WM 必须分离，不可合并**，理由（按决定性）：① 可用性契约不同——感知每 tick 必须有，WM 可旁路/超时即放行（[[non-blocking-recall]]），焊一起两契约打架；② 问题/消费者不同——感知答"现在是什么"喂 Agent，WM 答"做动作 a 会怎样"喂规划/护栏；③ 部署分体——视觉感知在云端、WM 在本地 3060，物理上无法合成一个网络；④ 模块化+可追溯——各自可换、故障可分辨是"看错"还是"算错"；⑤ 阶段——感知现在有、WM 以后才有，合并会阻塞感知落地。
3. **WM ≠ 视觉模型，"让 WM 承担视觉感知"=改名后的"本地多模态"。** 视觉成本在**编码器(重 ViT)**，动力学才是 WM 独有(轻)；把编码器塞进 WM 不省算力、且必须本地跑——就是你刚否掉的本地多模态。**共享编码器是"未来真上了本地视觉"才兑现的优化，不是现在**。
4. **训练料闭合**：符号 WM 的训练对 `(belief_t, action_t) → (belief_{t+1}, outcome)` 正好是数据飞轮的 `InteractionEpisode`（§五）——**你采的东西正好训你要本地部署的东西**。
5. **3060 6GB 能跑什么**（实测级）：✅ DreamerV3-small / TD-MPC2(5M, 现成权重) / SmolVLA 推理(紧)；❌ π0(>8GB)、OpenVLA 7B、GR00T、Cosmos(2B=32GB)。**第一个实验**：state-based TD-MPC2/Dreamer 玩具任务建立 latent 动力学+MPC 直觉。

## 四、VLA 落地结论

详见 [VLA 调研](vla-research-2026-06-28.md) 与附录 B。决策要点：

- **现阶段"埋接缝"而非"试跑"。** VTuber 阶段的动作 = 表情/手势/Live2D/VRChat OSC，**不是机器人关节空间**；严肃 VLA 在此错配，且没有机器人硬件 = 采不到演示数据（VLA 门槛 80% 在数据）。
- **VTuber 阶段动作走 LLM 出子目标 + 确定性动作库**（Voyager 范式的 B/C 模式），不引入端到端 VLA。
- **真机阶段**：本地小脑 = 蒸馏小 VLA（SmolVLA/NanoVLA）或 Jetson 级 GR00T；云脑-本地分工照 AWS Strands+GR00T 参考架构，**接口 = 自然语言/结构化子目标**。
- **VLA 生态 100% 是 Python**，与 TS/Node 主栈以 **sidecar（gRPC/HTTP）** 共存（LeRobot policy-server 即此模式），独立崩溃不拖垮主栈，符合优雅降级。

## 五、分阶段数据飞轮（VTuber/VRC 的真正目的）

VTuber/VRChat **不是终点，是数据飞轮**——目的是积累真实人类交流数据。但数据分三层，各喂不同的东西，**边界必须诚实**：

| 层 | 采到什么 | 喂给谁 | 这阶段能拿到吗 |
|----|---------|--------|---------------|
| **A 对话/社交** | 轮次、打断、附和时机、情绪轨迹、主动发话成败、关系演化 | Agent 大脑 + 社会世界模型 + 主动性/appraiser 小策略 | ✅ ROI 最高 |
| **B 具身-非语言** | 全身追踪姿态、共话手势、眼神、社交距离(proxemics) | 社会世界模型非语言通道 + Avatar 驱动 → 远期机器人社交行为 | ⚠️ 仅 VRC/Avatar 阶段能采（需 VR 追踪），这是 VRC 比纯语音 app 多出来的独特价值 |
| **C 操作/电机动作** | (视觉,指令,机器人轨迹) 三元组 | VLA 小脑 | ❌ **采不到**——VRC"物理"非真实动力学，Avatar 不操作真实物体 |

**最需点破**：VTuber/VRC **不替你解决 VLA 的动作数据问题（C）**，它是喂大脑(A)和社会非语言层(B)的飞轮。

**金矿 = 结果/奖励信号**：长期部署的陪伴体能采到"这次交互好不好"的纵向信号（停留时长/回访/情绪变化），公开数据集普遍缺这个。有了带结果标签的 episode，才能学 chat-A 已存在的小策略（主动时机/附和门控/EOU 阈值/appraiser），**不需要动 LLM**。

**基建复用**：[[voice-traceability]] 的 `CHAT_A_VOICE_TRACE` + `SqliteVoiceTraceSink` 就是 episode 语料库雏形，升级成多模态对齐的 `InteractionEpisode` 即可。

**隐私约束**：自己与小雪的交互可随便采；VRChat 他人/直播观众 = 录第三方，涉知情同意/平台条款，**第三方默认只存匿名行为统计，不存原始音频/身份**。

## 六、现在就埋的四个 Port（零成本，设计阶段）

符号边界一律用**自然语言/结构化意图**，不押注 latent（理由见 §7 对文档(7)的审查）。

| Port | 定义 | 对齐开源范式 | 硬约束 |
|------|------|-------------|--------|
| **PerceptionPort** | 多模态感知入口 | GR00T 的 VLM backbone | 现有语音管线是其子集 |
| **PlannerPort** | Agent 大脑出口：LLM 规划 + 技能库 | **Voyager 范式**、Strands | 与已有 tool-use/MCP 分层对齐 |
| **WorldModelPort** | 状态+候选动作 → 预测后果+风险评分。day1 可空实现/直通 | V-JEPA 2-AC、World-Env、FLARE | **必须异步旁路+可降级，超时即放行**，绝不焊进首字延迟（复用[[non-blocking-recall]]） |
| **ActionPort** | 意图/子目标 → 动作指令 | LeRobot 硬件抽象、GR00T VLA 接口 | VTuber 期=OSC/Live2D 确定性；真机期=VLA sidecar |

**VTuber 阶段妙用**：WorldModelPort 此时装的不必是物理世界模型，可以是轻量**"社交后果预测器"**（"这句话会不会冒犯"）——**同一个 Port，远期换物理世界模型时编排不变**。

### 6.1 中心枢纽：BeliefState 信念黑板（四角色缺的那个"中间层"）

四个 Port 解决了"模块怎么连"，但没解决"信息在哪儿汇总"。补一个**中心化的信念/工作状态黑板**——所有角色只跟它对话（N×N 耦合 → N 对 1），它是"把信息综合起来"的那个中间层（呼应 F 盘文档(7) §3.2 的 JSON 世界状态构想，但**货币是符号、不是 latent**）。

```
BeliefState(符号化·可追溯)
  perception:  { 谁在说话, 环境标签, 场景摘要(API低频), ... }   ← PerceptionPort 写
  social:      { user.PAD, 关系状态, 意图猜测, opennessToTalk }  ← 社会世界模型写(已有雏形)
  self:        { 小雪情绪PAD, 能量, 当前姿态/朝向 }
  predictions: { feasibility, expected_outcome, 社交后果/风险 }  ← WorldModelPort 写
  intent:      { 当前 setpoint: approach/follow/..., 回合意图 }   ← Agent/PlannerPort 写
  + 全字段可打 trace，复用 vtrace 范式
```

- **货币 = 结构化符号，不是共享 latent**（否则重踩 symbol→latent 接地坑）。各模块保留自己内部的 latent，黑板只存人类可读、可追溯的综合信念。
- **与事件总线互补**：事件总线是**传输**(消息流动)，黑板是**状态**(综合信念持久化)；事件更新黑板，模块读黑板。chat-A 已有的 UserState/PAD/记忆是其雏形——正式收敛成一个中心信念态即可。
- **快环只读它的轻量切片**（setpoint + 本地感知），不被整张黑板拖慢。
- **"分离 ≠ 不综合"**：感知与 WM 分离（见 §3.3），但经此黑板综合——这才是干净、可追溯的"信息综合"。

## 七、运动与实时性（VRC 运动控制 + 编排极致响应）

让系统在 VRC 里"动"起来，是把纯语音陪伴推向具身的关键一步——**纯说话用不上世界模型/动作，一旦要动，三条线（世界模型/动作/数据飞轮）才同时被激活**。但要诚实划边界。

### 7.1 双系统分工（用户独立推出，正确）

```
Agent(云端, ~1Hz, 语义思考)：「她情绪低落 → 靠近、保持陪伴朝向」
        │ 下发 setpoint/意图（离散、结构化，不是 latent）
        ▼
本地快环(10-60Hz)：读本地感知 → 追踪 setpoint → 发 OSC /input/ 运动
        ↑ Agent 永不进控制环 → 运动不受云端 ~1s 延迟拖累
```

- **接口 = 离散意图/setpoint，不是 latent**（symbol→latent 接地现实没人做，见 §八外部审查）。Agent 设 envelope，反射在 envelope 内自治。
- 这就是 GR00T/Helix/π0 的 System2→System1，只是把 latent action token 换成可调试的结构化意图。

### 7.2 关键约束：现阶段多模态=API → 快道传感器只有音频/OSC，没有视觉

| 模态 | 本地快处理 | 能驱动的运动 |
|------|-----------|-------------|
| **音频**（VAD/能量/韵律/声源方向） | ✅ chat-A 已在本地做 | 朝声音转头、轻声就凑近、情绪高就活跃、随语速晃动 |
| **OSC 结构化**（自身状态/手势/世界事件） | ✅ | 临场感动作、姿态、手势同步 |
| **视觉**（场景/障碍/他人位置） | ❌ 多模态=API=慢（~1s 往返、按帧计费） | 空间导航/避障/视觉跟随 **→ 锁死，等本地视觉** |

**反应延迟地板 = 一次云端往返 ~0.5-1s**，且送帧频率被成本卡在 ~1-2Hz。**全程流式级联也搬不动这个地板**——streaming 降的是 time-to-first-token/吞吐平滑度，不是"世界发生事→看懂并反应"的环路延迟。TTS 确实不在反应关键路径上（用户判断对），但瓶颈本来就不是 TTS，是云端视觉那一跳。

### 7.3 运动四级（按 API-only 约束判定）

| 级别 | 内容 | 现阶段判定 |
|------|------|-----------|
| **L0 临场感** | 看向说话人、转身、晃动 | ✅ **现在就做**——纯 OSC/本地音频驱动，不碰多模态，性价比最高 |
| **L1 社交-空间** | 靠近/退开(proxemics)、跟随 | ⚠️ 降级：隔几秒粗调可以；平滑视觉跟随不行 |
| **L2 导航 / L3 RL-in-VRC** | 走到点/避障/把 VRC 当 Gym 训动力学 | ❌ 锁死到本地视觉之后 |

L0 + 粗 L1 正好采**数据飞轮 Tier B 非语言数据**（姿态/距离/朝向），纯语音 app 采不到。

### 7.4 编排层"极致响应" + 单调升级纪律 ⭐

**现阶段方案 = 把编排层的快速响应做到极致；未来接本地模型只会更快、永不倒退。** 大半是语音栈延迟隐藏打法的迁移：

| 招 | 做法 | 复用资产 |
|----|------|---------|
| 快道永不等慢道 | 本地反射立刻发，云端结果异步到了再修正 | [[non-blocking-recall]]+优雅降级 |
| 投机执行 | 不等云端裁决，先按本地先验动，回来再纠偏 | 范式同 backchannel |
| 动作分块+本地插值 | 云端~1Hz 出粗 chunk，本地 60Hz 插值平滑（运动版流式 TTS） | 流式 TTS 缓冲思路 |
| 延迟掩蔽行为 | 等云端时发"思考态"微动作（重心晃/瞥/"嗯"）掩盖往返 | backchannel 具身版 |
| 持续流式感知+滚动信念 | 维护永远最新的"场景信念"，云端流式更新，绝不冷取 | 连续流式 ASR 范式 |
| 预热常驻连接 | API 用常驻 WS、模型/TTS 预热，免冷启动 | TTS 预热已做 |
| 本地优先级仲裁 | 合并"快反射+慢意图"的仲裁器跑本地、瞬时 | 优先级回合调度已有 |
| 砍冗余跳 | 每多一跳多几百ms~几秒 | 教训：翻译通道 2 次 LLM 调用曾让音频慢几秒 |

**保证"未来只会更快"的三条纪律（单调上升不是自动的）**：
1. **接缝契约延迟无关**：PerceptionPort 语义="给我当前最新场景信念"、ActionPort="接收这个意图"，**不是**"阻塞直到取回"。API→本地只是信念更勤、控制环更紧，纯增益零重写。
2. **降级优先**：按最坏延迟设计（超时用本地先验）。更快的本地模型永不违反系统已容忍的假设 → 无倒退风险。
3. **行为跨延迟区间都正确**：别把时序焊死在"恰好 1s 才好看"，否则换快模型反而破。做成"快慢都对"，那时快=严格更好。

满足三条，"接本地只会更快"在数学上成立。

## 八、外部方案审查结论（F 盘两份文档）

主理人提供的两份与其他大模型讨论的方案：

- **文档(6)**（AI VTuber 全双工 + VRC/Live2D，Fay/ChatVRM 脚手架）：**整体低于 chat-A 现状**。其"认知世界模型/JSON 状态机" = chat-A 已有的社会世界模型 + PAD/UserState，chat-A 做得更细；力推的 **ChatVRM 已于 2025-05-27 归档停更**，且它本质是回合制、并非文档所称的"原生全双工+打断"（全双工 chat-A 已做掉）。**价值在重新定位为数据飞轮前端**（把人拉进来产生交互）。
- **文档(7) V2.0**（云端 Agent + 本地物理世界模型级联，6GB，Sim2Real）：**方向对、工程骨架好，但核心赌注有两个硬伤**：
  1. **押注 DreamerV3/TD-MPC2 当小脑 + "零样本 Sim2Real"** —— 这恰是该方法最脆弱处（实证零样本上真机系统性失败）。
  2. **"隐空间通信协议"（云端 LLM 直接吐世界模型 latent 目标）** —— 三个子代理独立查证：**symbol→latent 接地现实里没人真这么干**，全退化成"云端发自然语言子目标，本地当条件吃"。这是没落地的承重墙。
  - **值得保留**：云脑(S2)+本地小脑(S1)级联、"latent-only 不生成像素"、6GB 工程纪律（ONNX/TensorRT、ASR/TTS 压 CPU、C++ 热环 30-60Hz）、碰撞惩罚护栏、域随机化、技能库沉淀。
  - **调和**：保留(7)好骨架，但**小脑从 Dreamer 换成小 VLA 动作专家，世界模型退回"想象+护栏"本职，符号边界用自然语言**。

## 九、值得现在跟踪的开源锚点

| 角色 | 锚点 | License | 为什么 |
|------|------|---------|--------|
| VLA 枢纽 | **LeRobot**（HF, 25k★） | Apache-2.0 | 事实标准，托管 SmolVLA/π0/GR00T，自带 policy-server 异步 sidecar |

| Agent 大脑 | **Voyager 范式** | MIT 系 | LLM+技能库+自验证，不微调，与 tool-use/MCP 对齐 |
| 双系统范本 | **GR00T N1.x** | 代码 Apache-2.0 | System2 VLM + System1 动作专家开源标杆 |
| 融合参考 | **GigaBrain-0 / WorldVLA** | 开源权重 | 世界模型×VLA 融合的真代码（多数融合方案仍是期货） |

**跨报告共识**：① 2025-26 是"融合"不是二选一，但融合目前只是 benchmark +6~36% 非质变，且最热的闭环联合训练（World-VLA-Loop 等）全是"code in preparation"期货；② 6GB/树莓派重型 VLA/世界模型一个都跑不动，本地只做反应执行，重计算上云/留仿真；③ 没有"一键端到端开源具身栈"，靠工程拼装（LeRobot+GR00T+Isaac+自研编排）。
